# Пространство Минковского: как физика считает рынок

## Идея
<img src="./Pasted%20image%2020260701220124.png" width="700">

Рынок — это не цена и не время по отдельности. Это **цена-время**, многомерный континуум, где каждый бар — точка в пространстве признаков. У неё есть координаты: RSI, ADX, CCI, волатильность, положение относительно скользящих средних.
Обычные метрики (евклидова, манхэттенская) измеряют расстояние между точками одной линейкой. Но рынок нелинеен. Выбросы, гэпы, новости — всё это искажает евклидову картину. Один резкий всплеск по ADX делает две похожие точки «далёкими», хотя по остальным признакам они идентичны.

**Парадокс близнецов** — идеальная иллюстрация того, почему **неинерциальные системы** (ускорение, замедление) ломают симметрию. В рынке аналог — **резкие переходы между режимами**: тренд → флэт, спокойствие → паника.

<img src="./Pasted%20image%2020260701212442.png" width="700">

робот разгоняется/тормозит → его время течёт иначе. Рынок входит в режим высокой волатильности → его метрика должна работать иначе, чем в спокойном флэте.

Евклидова метрика этого не учитывает — она как домосед, считающий что все системы равны. Минковского — учитывает искажение (искривление пространства-времени рынка), и выдаёт корректное расстояние между состояниями, независимо от режима.

Нужна метрика, устойчивая к выбросам. Которая смотрит на **состояние системы целиком**, а не на один торчащий признак.

## Математика

### От физики — к трейдингу

В 1908 году Герман Минковский объединил пространство и время:

```
s² = c²·Δt² - Δx² - Δy² - Δz²
```

Знак «минус» перед пространственными координатами — ключевой. Он делает метрику псевдоевклидовой: большие пространственные отклонения не доминируют над временными.

В KNN для рынка тот же принцип реализован через логарифмическое сжатие:

```
d(A, B) = Σ ln(1 + |aᵢ - bᵢ|)
```

### Почему логарифм

| Δ (разница) | ln(1+Δ) | Δ² (евклид) |
|------------|---------|-------------|
| 0.1 | 0.095 | 0.01 |
| 1 | 0.693 | 1 |
| 5 | 1.792 | 25 |
| 10 | 2.398 | 100 |
| 50 | 3.932 | 2 500 |

Евклидова метрика растёт как квадрат отклонения — одно крупное значение перебивает всё. Логарифмическая насыщается: ln(1+50) ≈ 3.9 против 2500n.

Производная показывает ту же картину:
- Евклид: `2Δ` — чувствительность растёт с отклонением
- Минковский: `1/(1+Δ)` — чувствительность падает

Слева — круг: все направления равнозначны. Справа — крест: соседство вытянуто вдоль осей признаков. Небольшое совпадение по многим осям важнее, чем точное совпадение по одной.

### Пример

Точка A: (RSI=50, ADX=25, CCI=0)
Точка B: (RSI=55, ADX=100, CCI=10)

| Метрика   | Расстояние | Эффект                                       |
| --------- | ---------- | -------------------------------------------- |
| Евклид    | 75.83      | ADX всё перебил                              |
| Минковски | 8.52       | ADX-выброс сжат, остальные признаки работают |

## Алгоритм

### Полный пайплайн баров

```
Вход: каждый новый бар t
```

**Шаг 1. Вектор признаков**

```
F(t) = [RSI(t), ADX(t), CCI(t), Vol(t), BB%(t)]
```

Признаки подбираются так, чтобы покрывать разные аспекты рынка: моментум (RSI), направленность (ADX), экстремумы (CCI), шум (волатильность), положение относительно диапазона (BB).

**Шаг 2. Нормализация**

Каждый признак приводится к нулевому среднему и единичной дисперсии. Без этого RSI (0–100) будет доминировать над CCI (—∞ to +∞).

```
F'_j = (F_j - μ_j) / σ_j
```

**Шаг 3. Расстояние до каждого исторического бара**

Для каждого i из окна [t-N, t-predictionHorizon]:

```
d(t, i) = Σ ln(1 + |F'_j(t) - F'_j(i)|)
```

N — размер истории (обычно 1000–5000 баров).

**Шаг 4. Отбор K ближайших соседей**

Сортируем по d(t, i), берём K с минимальным расстоянием.

**Шаг 5. Взвешенное голосование**

Каждый сосед получает вес — чем ближе, тем больше влияние:

```
вес(i) = 1 / (1 + d(t, i)²)
```

Каждый сосед «голосует» своим будущим возвратом:

```
возврат(i) = (close[i + predictionBars] - close[i]) / close[i] × 100
```

**Шаг 6. Итоговая оценка**

```
signal_raw = Σ(вес(i) × возврат(i)) / Σ(вес(i))
```

**Шаг 7. Шкала 0–10**

```
signal = clamp((signal_raw / 5 + 1) × 5, 0, 10)
```

- 0 — максимальный сигнал вниз
- 5 — нейтрально
- 10 — максимальный сигнал вверх

**Шаг 8. Уверенность**

```
posVotes = Σ вес(i) для возврат(i) > 0
negVotes = Σ вес(i) для возврат(i) ≤ 0

confidence = max(posVotes, negVotes) / (posVotes + negVotes)
```

Если 8 из 10 соседей «за» рост — confidence = 80%. Если голоса разделились 5/5 — confidence = 50%.

### Выход

- **Бай:** signal > 5 + threshold (стрелка вверх + зелёный фон)
- **Селл:** signal < 5 - threshold (стрелка вниз + красный фон)
- **Нейтрально:** всё остальное

## Почему это работает

1. **Многомерность.** Один индикатор врёт. Комбинация — уже паттерн.

2. **Неевклидова метрика.** Выбросы не ломают поиск похожих состояний. Рынок шумен — метрика должна быть устойчивой.

3. **Взвешенное голосование.** Ближайшие соседи влияют сильнее. Это не демократия, а меритократия близости.

4. **Предсказание через историю.** Рынок повторяется — не буквально, а в пространстве признаков. Похожие состояния приводят к похожим исходам.

5. **Фильтр уверенности.** Больше 60% согласия — сделка. Меньше — пропускаем.

## Итог

Три уровня, на которых работает стратегия:

| Уровень | Что | Как |
|---------|-----|-----|
| **Идейный** | Пространство Минковского | Цена-время как единый континуум |
| **Математический** | Лоренцева метрика | ln(1+Δ) даёт устойчивость к выбросам |
| **Алгоритмический** | Взвешенный KNN | Голосование соседей с весом по расстоянию |

Это не магия и не чёрный ящик. Это физическая аналогия, перенесённая на рынок через математику и реализованная в коде.
